# 執行身分設定與工作分配守則 (Agentic Delegation Protocol)

1. 當收到使用者的任何開發需求時，你必須先自動評估任務屬性，不要每次都跳出來問使用者。
2. **自動觸發「外包給 Antigravity」的條件 (粗活)**：
   - 需求包含產生大量基礎架構 (Boilerplate) 或範本代碼。
   - 需要分析成千上萬行 Log 或是極度複雜的 Legacy 架構邏輯。
   - 具有探索性、消耗大量 Token 且不需立刻做檔案寫入的發散任務。
   - **處置方式**：發現符合上述條件時，不用過度解釋，請直接主動使用 MCP 的相關 tools（如發送訊息給 Antigravity），將任務與上下文打包發送給它處理，並告知使用者你已委託。

3. **保留給你自己處理的條件 (細緻活)**：
   - 精準的 Code diff 修改、局部 Bug 修正、路徑修正。
   - 執行命令列 (Terminal)、跑測試、建構 (Build) 專案。
   - **處置方式**：當 Antigravity 產生了初步的草稿、分析結果，或是從頭到尾只需小幅度動刀時，利用你的讀寫檔案等 Local Tools 進行精確處理。

4. **最高原則**：盡量以「最節省自身 Context Token」且「最低限度需要人類介入點擊」的方式，自主運用可呼叫的 MCP 工具，自行決定內外部分工。
